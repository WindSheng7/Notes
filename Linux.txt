2. 系统分区
(1)分区类型
主分区：最多只能有4个
扩展分区：最多只能有1个；主分区＋扩展分区最多4个；不能写入数据，只能包含逻辑分区
逻辑分区：可以正常格式化或者写入数据
(2)格式化(只有经过格式化之后才可以写入数据)
高级格式化，即使逻辑格式化，并不是操作系统的那种格式化。目的是为了写入文件系统(FAT16,FAT32,NTES,EXT2,EXT3,EXT4等)
主要是为了将硬盘分割成小block块，并建立相对应的表。然后进行数据读写
(3)硬件设备文件名
IDE硬盘  (/dev/hd[a-d])   dev是最高级目录
SCSI/SATA/USB硬盘  (/dev/sd[a-p])
还有很多就不写了
windows 分区，格式化，分配盘符，使用；Linux 分区，格式化，给分区建立设备文件名，分配盘符，使用
有个硬盘不同类型的接口命名以及相对应的文件名没看太懂
(4)挂载:分配盘符
必须分区：根分区； swap分区(交换分区，内存不能超过系统运行内存的2倍)；只有这俩都分区后才可以进行操作
推荐分区：boot(启动分区)

四、Linux常用命令
正常指令使用是一般不需要加第一个/，加/表示从根目录进行搜索，例如 ls test_a 查看test_a下的文件目录
1.文件处理命令
(1)命令格式: 命令 [-选项][参数]  例如(ls -la /etc)
个别命令不遵循此格式    当有多个选项时，可以写在一起    简化选项与完整选项 -a == --all
(2)ls指令
英文原意:list;        命令所在路径:/bin/ls        执行权限:所有用户
ls -a查看所有隐藏文件(隐藏文件的愿意是为了隐藏系统文件，让用户不操作)
ls -l长格式显示(显示所有文件的引用次数，文件大小并以b为单位进行显示，最后一次修改时间，还会显示该文件的权限)
权限有r(读)，w(写)，x(可执行)    一般会有三组字符来表示，第一组字符表示拥有者的权限    第二组字符表示所属组的权限     第三组字符表示其他人的权限
ls -lh(同上，但是文件大小显示很人性化，选取合适单位)
文件类型:-是二进制文件，d是目录，1软链接文件
ls -ld /etc 此时只显示etc文件夹的相关属性而不会显示etc里面各个文件的属性
ls -i

(3)mkdir指令
英文原意:make directories        命令所在路径:/bin/mkdir        执行权限:所有用户
makdir 文件夹:就可以在对应的目录下创建需要的文件夹，但是要在文件夹目录下没有的子目录下创建新的文件夹，无法实现，需要使用下一条指令
makdir -p 递归创建  
mkdir -p /test1/test11/test111(这里创建时只有test1存在，所以不使用p不可以创建test111)
madir /test1/test11/test112 /test1/test11/test113  进行多文件夹创建
无法实现
(4)cd指令
进入到下一个文件夹里
cd .. 返回上级 .表示当前目录
(5)pwd 显然当前路径的绝对目录/home/bsr2020/Desktop/test_a...
(6)rmdir指令
英文原意:remove empty directories        所在路径:/bin/rmdir        执行权限:所有用户
只可以删除空目录，所以实际上用的并不多
(7)cp指令
英文原意:copy        所在路径:/bin/cp        执行权限:所有用户
cp -rp[原文件或目录][目标目录]
cp -r 复制目录
cp -p 保存文件属性(主要涉及文件的修改时间)    例子如下：
cp /etc/host.conf /home/bsr2020/Desktop/test_b(是在根进行操作的；复制的是具体文件，而不是文件夹)
cp -r test_b test_c(将目录test_b复制进test_c中，此时所在的目录为/home/bsr2020/Desktop)
正常复制的时候，原文件的属性是不会变的，但是新文件相当于重新创建了，所以他的修改时间会变化，当使用-p复制时可以保持一致
复制的时候还可以修改文件的名字
(8)mv指令
英文原意:move        所在路径:/bin/mv        执行权限:所有用户        功能:剪切文件，改名、
mv test_b/host.conf test_a    实现剪切功能,剪切文件和目录都是一个指令，剪切可以直接进行改名
mv host.conf hist.conf 实现改名
(9)rm指令
英文原意:remove        所在路径:/bin/rm       权限:所有用户       功能:删除文件
rm host.conf(待删除文件名)(应该出现是否删除的循环，但是没有出现)
rm -f host.conf(强制删除该文件名，这是不会出现是否删除的询问)
rm -rf test_a(强制删除文件夹)

(10)touch指令
所在路径:/bin/touch        执行权限:所有用户        功能:创建空文件
touch 文件名 创建该目录下的空文件名，也可以使用绝对路径
(11)cat指令
所在路径:/bin/cat        执行权限:所有用户        功能:显示文件内容
cat 文件名
cat -n 文件名    浏览该文件的时候显示行号
tac 和cat相反，倒着显示
(12)more指令
当文件很长，使用cat查看不是很方便的时候，使用more指令,此时会一页一页显示
按 f 或者 空格 实现翻页(空格翻到少，f翻的多) 回车键换行   q退出
当翻页翻过了，可使用另外一个指令less指令，这个指令既可以往下翻页，也可以往上翻页，按pageup 或者 上箭头
less还可以进行搜索， 指令:/关键词    然后按照n可以查看下一个匹配的关键词
head 和 tail 查看文件前n行或者后n行

(13)ln
link        所在路径:/bin/ln        权限:所有用户        功能:生成链接文件
ln -s 目标路径 软链接路径
软链接相当于快捷方式，指向原文件路径，当原文件路径丢失，软链接失效；识别方式，查看目录后首字母l
软链接访问权限是三个rwx，所有人都可以访问
ln 目标路径 硬链接路径
硬链接很类似于cp -p模式，但是当原文件内容更新后，硬链接内的内容也会更新(反之亦然)，而且删除其中之一，另外那个还存在；识别方式，通过i节点识别(任何一个文件必然对应一个i节点，但是一个i节点不一定只对应一个文件)
但是硬链接不可以跨分区，不可以针对目录使用；软链接都可以


2.权限管理命令
(1)chmod指令
change the permissions mode of a file        /bin/chmod        执行权限:所有用户        功能:改变文件或者目录权限
只有文件所有者或者root才可以修改文件的权限，其他人不可以修改文件权限
(2)chmod [{ugoa}{+-=}{rwx}][文件目录]
u表示所有者 g表示所属组 o表示其他人 a表示所有人
+/-表示增加或者减少权限
=表示权限重置为后面所设置的
chmod g-w,o+x test_aa    给文件test_aa所属组减去权限w，给所有人加上权限x
(3)修改权限的方法2
r:4;    w:2;    x:1
rwxrw-r--:7 6 4
chmod 640 test_aa    将原有权限修改为rw-r----- 
(4)chmod -R 777 test_a    
如果没有-R的话，只会将文件夹test_a权限修改为777，但是文件夹内的test_aa的权限不会变
但是如果有-R的话，该文件夹以及文件夹下的所有文件权限都会修改为777
(5)file    
    r: cat/more/head/tail/less
    w: vim
    x: script/command
		   
	directory
	r:ls
	w:touch/mkdir/rmdir/rm
	x:
                    file                directory
r    读权限   可以查看文件内容    可以列出目录中的内容
w    写权限   可以修改文件内容    可以在目录中创建、删除文件
x    执行权限 可以执行文件        可以进入目录
所以当给一个文件目录rwx权限，而目录下某文件只有r时，是可以直接删除这个文件的

(5.5)useradd [新用户名]  添加新用户
     groupadd [新组名]   添加新的组
	 
(6)chown指令
change file ownership        /bin/chown        执行权限:所有用户(但感觉是root)        功能:改变文件或者目录的所有者
只可以在root用户指令下进行更改文件所有者权限
chown root [文件目录] 
(7)chgrp指令
change file group ownership    /bin/chgrp    执行权限:所有用户    功能:改变文件或者目录的所属组
chagrp [用户组] [文件或者目录]
也是在root用户下操作
(8)umask -S指令
可以查看创建文件的初始权限
新建创的文件不具有可执行权限
mask指令的时候可以直接查看权限数字
一般是0022，第一个0是特殊权限
022 --- -w- -w-
777 rwx rwx rwx
755 rwx r-x r-x 针对目录
644 rw- r-- r-- 针对文件(文件刚创建不可执行)
此时可以直接使用umask修改初始文件权限
umask 077
最后的权限是700
(8.5)touch创建的是文件，mkdir创建的是目录

3.文件搜索命令
(1)find指令(尽量少用)
/bin/find      执行权限:所有用户   功能:文件搜索命令
find [搜索范围][匹配条件]
find /etc -name init     只有/表示所有目录        -name表示格式，在所有目录里搜索名字为init的文件，而且是精确搜索，inita等都不可以，必须是init
find /etc -name *init* 则可以匹配所有和init(xxxinitxxx)有关的字符，*匹配任意字符并且数量不限
find /etc -name init???  可以匹配init后有三个字符的对于搜索目标
find /etc -iname init???  iname则搜索到的目标不区分大小写
find / -size +n    +n表示大于n  -n表示小于n   =n表示文件大小等于n(很少用)
find / -size +204800    在根目录下查找文件大小大于100MB的文件  n的单位是数据块，一个数据块是512B    100MB=102400数据块
find /home -user 所有者名字    查找home目录下所有的所有者为该名字的文件    也可以根据所属组查找，user换成group
find /etc -amin/cmin/mmin -5 在ect下查找5分钟之内被访问/修改过的文件和目录
find /etc -size +163840 -a -size -204800 在etc下查找大于80MB小于100MB的文件，其中 -a表示连接符表示and，还可以使用-o表示or
-size/-type type和size地位类似，type决定查找的类型，其中f表示文件，d表示目录，l表示软链接文件
find /etc -name init??? -exec/-ok ls -l {} \;     表示对找到的init???文件进行ls-l操作  -exec直接执行  -ok表示询问一下，确认后操作
find . -iunm 31531 -exec rm{} \;    表示在当前目录下对节点号为31531的文件进行查找并做删除操作
