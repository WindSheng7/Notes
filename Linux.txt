2. 系统分区
(1)分区类型
主分区：最多只能有4个
扩展分区：最多只能有1个；主分区＋扩展分区最多4个；不能写入数据，只能包含逻辑分区
逻辑分区：可以正常格式化或者写入数据
(2)格式化(只有经过格式化之后才可以写入数据)
高级格式化，即使逻辑格式化，并不是操作系统的那种格式化。目的是为了写入文件系统(FAT16,FAT32,NTES,EXT2,EXT3,EXT4等)
主要是为了将硬盘分割成小block块，并建立相对应的表。然后进行数据读写
(3)硬件设备文件名
IDE硬盘  (/dev/hd[a-d])   dev是最高级目录
SCSI/SATA/USB硬盘  (/dev/sd[a-p])
还有很多就不写了
windows 分区，格式化，分配盘符，使用；Linux 分区，格式化，给分区建立设备文件名，分配盘符，使用
有个硬盘不同类型的接口命名以及相对应的文件名没看太懂
(4)挂载:分配盘符
必须分区：根分区； swap分区(交换分区，内存不能超过系统运行内存的2倍)；只有这俩都分区后才可以进行操作
推荐分区：boot(启动分区)

四、Linux常用命令
正常指令使用是一般不需要加第一个/，加/表示从根目录进行搜索，例如 ls test_a 查看test_a下的文件目录
1.文件处理命令
(1)命令格式: 命令 [-选项][参数]  例如(ls -la /etc)
个别命令不遵循此格式    当有多个选项时，可以写在一起    简化选项与完整选项 -a == --all
(2)ls指令
英文原意:list;        命令所在路径:/bin/ls        执行权限:所有用户
ls -a查看所有隐藏文件(隐藏文件的愿意是为了隐藏系统文件，让用户不操作)
ls -l长格式显示(显示所有文件的引用次数，文件大小并以b为单位进行显示，最后一次修改时间)
ls -lh(同上，但是文件大小显示很人性化，选取合适单位)
文件类型:-是二进制文件，d是目录，1软链接文件
ls -ld /etc 此时只显示etc文件夹的相关属性而不会显示etc里面各个文件的属性
ls -i

(3)mkdir指令
英文原意:make directories        命令所在路径:/bin/mkdir        执行权限:所有用户
makdir 文件夹:就可以在对应的目录下创建需要的文件夹，但是要在文件夹目录下没有的子目录下创建新的文件夹，无法实现，需要使用下一条指令
makdir -p 递归创建  
mkdir -p /test1/test11/test111(这里创建时只有test1存在，所以不使用p不可以创建test111)
madir /test1/test11/test112 /test1/test11/test113  进行多文件夹创建
无法实现
(4)cd指令
进入到下一个文件夹里
cd .. 返回上级 .表示当前目录
(5)pwd 显然当前路径的绝对目录/home/bsr2020/Desktop/test_a...
(6)rmdir指令
英文原意:remove empty directories        所在路径:/bin/rmdir        执行权限:所有用户
只可以删除空目录，所以实际上用的并不多
(7)cp指令
英文原意:copy        所在路径:/bin/cp        执行权限:所有用户
cp -rp[原文件或目录][目标目录]
cp -r 复制目录
cp -p 保存文件属性(主要涉及文件的修改时间)    例子如下：
cp /etc/host.conf /home/bsr2020/Desktop/test_b(是在根进行操作的；复制的是具体文件，而不是文件夹)
cp -r test_b test_c(将目录test_b复制进test_c中，此时所在的目录为/home/bsr2020/Desktop)
正常复制的时候，原文件的属性是不会变的，但是新文件相当于重新创建了，所以他的修改时间会变化，当使用-p复制时可以保持一致
复制的时候还可以修改文件的名字
(8)mv指令
英文原意:move        所在路径:/bin/mv        执行权限:所有用户        功能:剪切文件，改名、
mv test_b/host.conf test_a    实现剪切功能,剪切文件和目录都是一个指令，剪切可以直接进行改名
mv host.conf hist.conf 实现改名
(9)rm指令
英文原意:remove        所在路径:/bin/rm       权限:所有用户       功能:删除文件
rm host.conf(待删除文件名)(应该出现是否删除的循环，但是没有出现)
rm -f host.conf(强制删除该文件名，这是不会出现是否删除的询问)
rm -rf test_a(强制删除文件夹)

(10)touch指令
所在路径:/bin/touch        执行权限:所有用户        功能:创建空文件
touch 文件名 创建该目录下的空文件名，也可以使用绝对路径
(11)cat指令
所在路径:/bin/cat        执行权限:所有用户        功能:显示文件内容
cat 文件名
cat -n 文件名    浏览该文件的时候显示行号
tac 和cat相反，倒着显示
(12)more指令
当文件很长，使用cat查看不是很方便的时候，使用more指令,此时会一页一页显示
按 f 或者 空格 实现翻页(空格翻到少，f翻的多) 回车键换行   q退出
当翻页翻过了，可使用另外一个指令less指令，这个指令既可以往下翻页，也可以往上翻页，按pageup 或者 上箭头
less还可以进行搜索， 指令:/关键词    然后按照n可以查看下一个匹配的关键词
head 和 tail 查看文件前n行或者后n行

(13)ln
link        所在路径:/bin/ln        权限:所有用户        功能:生成链接文件
ln -s 目标路径 软链接路径
软链接相当于快捷方式，指向原文件路径，当原文件路径丢失，软链接失效；识别方式，查看目录后首字母l
软链接访问权限是三个rwx，所有人都可以访问
ln 目标路径 硬链接路径
硬链接很类似于cp -p模式，但是当原文件内容更新后，硬链接内的内容也会更新(反之亦然)，而且删除其中之一，另外那个还存在；识别方式，通过i节点识别(任何一个文件必然对应一个i节点，但是一个i节点不一定只对应一个文件)
但是硬链接不可以跨分区，不可以针对目录使用；软链接都可以
